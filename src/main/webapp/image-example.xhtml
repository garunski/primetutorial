Image Examples with SCSS Aliases

This page demonstrates how to use image aliases in SCSS files. The Vite configuration includes a resolve/alias for images that allows you to reference images using the @images path in your SCSS files.

How it works

In the Vite configuration (vite.config.js), we've added the following alias:

resolve: {
  alias: {
    '@images': path.resolve(__dirname, 'src/images')
  }
}

This allows you to reference images in your SCSS files using the @images path prefix with nested folders:

background-image: url('@images/common/sample-image.png');

Vite will resolve this path correctly during the build process and preserve the nested folder structure in the output directory.

Example 1: Background Image

Using background-image with the @images alias:

.example-image-background {
  background-image: url('@images/common/sample-image.png');
}

Example 2: Pseudo-element

Using a ::before pseudo-element with the @images alias:

This text has an image
&::before {
  background-image: url('@images/common/sample-image.png');
}

Example 3: Card with Image

A card component with a background image using the @images alias.

.image-card {
  background-image: url('@images/common/sample-image.png');
}

Example 4: Multiple Images

Using multiple background images with different paths:

background-image:
  url('@images/common/sample-image.png'),
  url('@images/icons/example-icon.svg');
